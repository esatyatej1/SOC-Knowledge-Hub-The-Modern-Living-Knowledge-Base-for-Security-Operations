# Contributing to SOC Knowledge Hub

Thank you for your interest in contributing to the SOC Knowledge Hub! We want to make it easy for the community to contribute and help us maintain an authoritative, structured, and modern SOC knowledge base.

## Our Philosophy
We are not just a list of links. We aim for:
- **Structure**: Every entry must follow our metadata template.
- **Practicality**: Focus on things an analyst can use *today*.
- **Freshness**: Regularly validate links and content.
- **Role-based**: Help specific roles find what they need.

## How to Contribute

### 1. Adding a Resource
If you want to add a tool, blog, guide, or template, please use the [Resource Submission Template](.github/ISSUE_TEMPLATE/add-resource.md).
Ensure you fill out all the metadata fields:
- Title
- Category
- Type
- Summary
- Why it matters
- Best for
- Difficulty
- Scope (Vendor-neutral/specific)
- Trust Level
- Link

### 2. Updating Content
If you find a broken link or outdated information, please open an issue or submit a Pull Request.

### 3. Improving Documentation
We welcome improvements to the core chapters in the `docs/` folder.

## Submission Guidelines
- Use clear, professional language.
- Ensure links are direct to the source.
- Avoid promotional content for paid products unless they are industry standards.
- Follow the existing folder structure.

## Style Guide
- Markdown headers should be hierarchical (H1, H2, H3).
- Use bold text for key terms.
- Use tables for comparative data.
- Ensure all images have descriptive alt text.

## Pull Request Process
1. Fork the repo.
2. Create a new branch (`feat/add-sigma-resources`).
3. Commit your changes.
4. Open a PR with a description of your changes.
5. All PRs must pass the CI checks (Markdown linting).
