# SOC Fundamentals

## What is a Security Operations Center (SOC)?
A Security Operations Center (SOC) is a centralized unit that deals with security issues on an organizational and technical level. It comprises a team of cybersecurity professionals responsible for monitoring, detecting, analyzing, and responding to cyber threats.

## Core Functions of a SOC
1. **Continuous Monitoring:** 24/7/365 observation of logs, network traffic, and endpoint activity.
2. **Incident Detection:** Identifying suspicious behavior using SIEM, XDR, and behavioral analysis.
3. **Incident Response:** Taking immediate action to contain, eradicate, and recover from security incidents.
4. **Threat Hunting:** Proactively searching for threats that have bypassed existing security controls.
5. **Log Management & Compliance:** Collecting, centralizing, and storing logs for auditing and analysis.
6. **Vulnerability Management:** Identifying and prioritizing weaknesses before attackers can exploit them.

## SOC Generations
- **1st Gen:** Point solution monitoring (Log-based).
- **2nd Gen:** SIEM-centric monitoring.
- **3rd Gen:** Logic-based, SOAR, and Big Data driven.
- **4th Gen (Current):** AI-augmented, Cloud-Native, and Intelligence-Led operations.

## Key SOC Roles
- **L1 Analyst (Triage):** Focuses on initial alert review and basic containment.
- **L2 Analyst (Responder):** Investigates complex alerts and executes remediation.
- **L3 Analyst (Hunter/Expert):** Deep forensics, advanced threat hunting, and malware analysis.
- **SOC Manager:** Oversees operations, strategy, and stakeholder communication.
- **Detection Engineer:** Builds and tunes detection logic to reduce noise.

## Frameworks to Know
- **NIST Cybersecurity Framework (CSF):** Identify, Protect, Detect, Respond, Recover.
- **MITRE ATT&CK:** Mapping adversary tactics and techniques.
- **Cyber Kill Chain:** Understanding the stages of an attack.
- **Unified Kill Chain:** A modernized version of the kill chain.
