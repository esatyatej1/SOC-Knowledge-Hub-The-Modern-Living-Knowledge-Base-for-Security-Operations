# L1 Analyst Learning Path

## Role Description
The L1 Analyst is the first line of defense. The primary goal is to triage incoming alerts, perform initial analysis, and either resolve or escalate the incident.

## Required Skills
- **Network Basics:** Understanding TCP/IP, DNS, HTTP/S, and common ports.
- **OS Proficiency:** Navigating Windows Event Logs and Linux /var/log/.
- **SIEM Basics:** Searching logs, using filters, and understanding alert summaries.
- **Security Basics:** Understanding the Cyber Kill Chain and common attack vectors.

## Monthly Goals
### Month 1: The Basics
- Learn how to read raw logs (PCAP, Event Logs).
- Understand the organization's network architecture.
- Master the triage workflow.

### Month 2: Tool Mastery
- Learn advanced search queries in the SIEM.
- Understand EDR alerts and how to isolate a host.
- Learn how to identify false positives.

### Month 3: Independent Triage
- Handle 90% of L1 alerts without escalation.
- Suggest 1 tuning recommendation for a noisy rule.
- Participate in an Incident Response simulation.
